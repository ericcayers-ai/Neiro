All-in-One Source Separation & Symbolic Transcription Application RoadmapThis roadmap establishes a decoupled, graph-based architecture designed for high-throughput local audio processing. By segregating individual deep learning nodes from the runtime orchestration engine, the application remains completely format-agnostic and inherently future-proof against shifting neural framework standards.Phase 1: Core Processing Foundations & Memory OrchestrationThis layer normalizes incoming wave files and manages physical hardware allocations before any deep inference operations take place.Dynamic Data Ingest Pipeline: An internal audio ingestion tool that reads arbitrary media files into memory-mapped raw floating-point tensors. The framework dynamically switches sample-rate lanes based on structural target node profiles—allocating down-sampled tracks for sequence language tokens and high-fidelity tracks for multi-band wave separation.Asynchronous Processing Graph: Operates via a directed acyclic graph (DAG) task engine. Every separation or transcription pass runs as an isolated execution thread, transmitting multi-channel tensors safely down the data stream.VRAM Context Virtualization Manager: An autonomous physical hardware sentinel that continuously tracks video memory ceilings. Before loading highly intensive transformer-based token decoders, it forces an explicit dump of separation weights out of the cache and calls garbage collection routines to prevent CUDA memory faults.Phase 2: Dual-Tier Source Separation EngineThe separation module isolates structural components out of dense arrangements, operating as a standalone feature or an adaptive front-end pre-filter for transcription.Dual-Path Splitting Framework:Single-Pass Path: A lightweight, low-latency lane optimized for fast extractions, everyday listening tracks, or real-time local processing.Studio Ensemble Path: A deeper multiband crossover configuration utilizing coordinate-based position embeddings and weighted linear energy averaging to output clean, highly isolated instrument stems.Mathematical Residual Extraction Layer: Built on raw phase inversion matrix math. Instead of wasting processing cycles forcing deep networks to identify rare synthesizers or transient sound effects, the pipeline extracts primary structural tracks and phase-inverts them against the absolute source file. The arithmetic remainder cleanly isolates the background texture layer without introducing digital compression artifacts.Phase 3: Modular Symbolic Notation EngineThis engine translates multi-instrument audio arrays into precise symbolic musical parameters, separating performance dynamics from grid notation layouts.Polyphonic Event Interval Decoders: Non-hierarchical transformer architectures that track explicit time-interval parameters rather than calculating fixed time frames. This enables highly accurate micro-timing detection and micro-tonal velocity tracking.Generative Diffusion Quantizers: Uses audio-conditioned diffusion steps to track chaotic, non-harmonic transient blocks like complex percussion grooves. This reconstructs complex arrangements by treating midi events as an iterative denoising process.Sequence-to-Sequence Multi-Task Tokenizers: Autoregressive attention frameworks that parse polyphonic sound arrays directly into symbolic strings representing chord progressions, master downbeats, pitch states, and mechanical variations.Phase 4: Auto-Split Orchestration & Track AggregationThis optimization pipeline filters complex mixes to bypass the precision degradation inherent in dense multi-instrument decoding.Conditional Routing Infrastructure: When the user enables the auto-split optimization mode, the application intercept layer blocks global multi-instrument models. It performs deep background source separation on the master mix, routes each isolated instrument track to its highly specialized mono-instrument translation model, and prevents cross-frequency masking errors.Temporal Timeline Compiler: An aggregation layer that functions as an event synchronizer. It catches parallel symbolic data streams coming out of the specialized mono nodes, matches absolute master clock offsets, routes specific instrument families to independent data tracks, and generates a unified multi-track output format.Downstream Dynamics Optimization: A post-processing pass that measures the compiled timeline parameters alongside the unseparated performance audio file to dynamically balance velocities, matching the natural human playing expression.Phase 5: Modular Abstraction Layer & Local Plugin SystemTo prevent the application code from breaking as open-source developments advance, all network variants are abstractly managed.Unified Model Interface Class: Every target neural network repository or local model fork is wrapped inside a rigid, uniform class template mapping parameters cleanly to universal entry methods.Dynamic Manifest Registration: The application initializes modules via an open, local JSON configurations layout. Users can expand the interface, swap underlying engine weights, or create new structural ensembles by dropping resources into a local folder without modifying the core backend script.ReferencesCore Frameworks & System UtilitiesFFmpeg Multimedia Architecture.
