# Support

Neiro is a local, open-source audio worksuite. Support happens in public so
others can benefit.

## Where to get help

1. **Documentation** — start with [README.md](README.md) and the docs under
   [`docs/`](docs/). Architecture, models, performance, sessions, plugins, and
   evaluation are covered there.
2. **GitHub Discussions / Issues** — use the issue templates for bugs, feature
   requests, model/adapter problems, performance regressions, accessibility, and
   documentation gaps.
3. **Security** — private disclosure only; see [SECURITY.md](SECURITY.md). Do
   not file public issues for vulnerabilities.

## What to include

- Neiro version (`neiro --version` or About in the UI)
- OS and hardware (CPU / GPU / VRAM)
- Whether neural extras are installed (`pip show audio-separator demucs …`)
- Exact command or UI path, and progress / error text (no magical paraphrases)
- Whether the failure reproduces on the DSP floor (no model downloads)

## Scope

Maintainers prioritize reproducible bugs, regressions against the golden
synthetic evaluation suite, accessibility blockers, and model-license / safety
issues. Feature requests are welcome but are scheduled against
[`roadmap.md`](roadmap.md).

## Community standards

Participation is governed by [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
